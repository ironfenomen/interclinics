# Интерклиника — безопасная локальная разработка

Краткие правила, чтобы правки не ломали dev и не запускали цикл «почини localhost».

## Обязательный порядок после содержательной правки

1. Изменить код.
2. **`npm run check`** — типы + ESLint (быстро, до открытия браузера).
3. Перед тем как считать задачу **закрытой** (мердж/переключение контекста): **`npm run verify`** — типы + линт + production build.
4. Локальный просмотр: только **`npm run dev`** (не плодить `next dev` вручную в других терминалах без остановки предыдущего).

Команда **`npm run dev:safe`** = сначала `check`, затем `dev` (если не прошёл check — сервер не стартует).

## Dev-сервер

- **Основной запуск:** `npm run dev` (фиксированный порт по умолчанию `3000`, перед стартом освобождается этот порт).
- **Не** запускать несколько `next dev` на одном порту — второй инстанс мешает первому. Один терминал = один dev.
- Странные runtime / webpack / «битый» HMR: **`npm run dev:clean`** (очистка `.next` + старт), при необходимости **`npm run dev:stop`** и снова **`npm run dev`**.

## Границы кода

- Меняя импорты/экспорты, помнить про **Server vs Client** (`'use client'`, серверные компоненты, `next/navigation`).
- Не оставлять проект в состоянии «в браузере вроде открылось» — без **`npm run check`** / **`verify`** задача не считается сделанной.

## Скрипты проверок

| Команда | Назначение |
|--------|------------|
| `npm run check:types` | TypeScript (`tsc --noEmit`) |
| `npm run check:lint` | ESLint (`next lint`) |
| `npm run check:build` | `next build` |
| `npm run check` | types + lint |
| `npm run verify` | types + lint + build |

## Для ассистентов (Cursor)

После правок в репозитории: выполнить **`npm run verify`** (или минимум **`npm run check`**), пока не проходит — задачу не завершать.
